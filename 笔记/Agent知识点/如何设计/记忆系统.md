# Agent的记忆和普通对话历史有什么区别？
普通对话历史就是把用户说了什么，AI回答了什么直接全部存下来，但是Agent的记忆更加复杂和结构化。

Agent记忆系统会对历史进行压缩、摘要，以保留最关键的信息。不止当前对话，还包括跨会话的长期信息，比如用户偏好、上次完成的任务、重要事实等。这些信息会长久保存。

记忆的检索方式也不一样，对话历史是按时间顺序的，Agent的记忆则是按照、语义相关性检索的。

# Agent的短期记忆实现方法：
1.滑动窗口：保留最近N轮对话，超出则丢弃
2.摘要压缩：把部分原始记录用大模型压缩成摘要
3.实际应用通常结合两种方式，保留最近3-5轮原始对话，再加一个之前对话的摘要

# Agent长期记忆存储方案
- 向量数据库：Agent会把需要记住的信息转化为向量存贮减去，检索时通过语义相似度来匹配。适合存储大量非结构化文本，比如用户的历史对话、文档、资料等（如Pinecone、Milvus、Weaviate）
- 键值存储：适合存储结构化简单数据，如用户ID对应的姓名、偏好设置、常用地址等。读写速度快，但是不适合存储大量文本
- 长期记忆主要存储什么？：用户基本资料、偏好、国王任务的完成情况、重要约定和承诺、对话里提炼出来的关键信息
# Agent怎么从记忆里找到相关信息？
人提问---（Embedding）--->向量----（和记忆库里的向量做语义相似度比较）---->找出最接近的几条记忆

# 为什么要区分长期短期记忆，各解决了什么问题？
- 短期记忆：解决当前任务上下文问题
- 长期记忆，解决 跨会话个性化 问题
- 两者区别：
     - 时间维度：短期记忆保留当前会话，长期记忆是跨会话持久化的
     - 容量维度：短期记忆容量有限，长期记忆可以存储很多
     - 内容维度：短期记忆存的是原始对话和上下文，长期记忆存的是经过提炼的偏好和事实
    只靠短期记忆无个性化，只靠长期记忆上下文会被稀释

# 记忆太多会有什么问题？怎么优化？
- 成本问题：检索计算成本变高，请求带进prompt的token变多，token成本直线上升
- 相关性噪声：不相关的东西会
