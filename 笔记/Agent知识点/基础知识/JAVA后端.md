JAVA Gateway
^javagateway
在微服务架构中，**Java API 网关**（如 Spring Cloud Gateway）是系统的“**单一入口**”和“**智能交通枢纽**”：它通过响应式编程高效地处理所有外部请求，利用断言（Predicates）**匹配路由，并通过**过滤器（Filters）在请求前后执行鉴权、限流、日志监控及协议转换，从而将复杂的横切关注点与业务微服务解耦，确保系统的高可用与安全性。

**API 网关**是微服务架构中的**统一入口**。
它的核心作用是将**非业务逻辑**（如鉴权、限流、日志和路由转发）从各个微服务中剥离出来，进行**集中式管理**。

这样可以实现业务逻辑与系统治理的**解耦**：客户端不再直接与多个微服务通信，而是只对接网关，由网关根据配置将请求精准地**路由**到后端服务，从而保障了整个系统的安全性与可维护性。
