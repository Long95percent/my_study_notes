# 是什么？用处？
提示词工程的目的是最大化AI模型的性能，让它产出更符合需求、高质量的输出。提示词工程可以提高准确性、节省时间、解锁能力、降低成本、解锁能力。对于开发者而言，它是模型配置的接口，通过精心设计的提示词比如说system prompt,可以定义AI助手的角色，行为准则和知识范围。**影响应用效果和成本**：好的提示词能用更短的交互、更低的 API 调用成本，获得更优的结果。

## 提示词的基本结构
- System Prompt:设定AI身份、规则和行为准则，在对话开始钱生效
- User Prompt:用户每次发出的消息，提出任务或者问题
- Assistant:AI的回复，也可以预填内容，让AI从那里继续
## Token与上下文窗口
经验公式：**1000 Token ≈ 750 个英文单词 ≈ 500 个中文汉字**

System Prompt:精炼优先，去掉重复说明，核心规则控制在 500 Token 以内
输入长文档:先摘要再输入，或使用 RAG（检索增强）方式只传入相关片段
多轮对话:历史消息会累积消耗 Token，长对话要注意定期"重置"或压缩历史
API 开发:输入 Token + 输出 Token 都计费，输出通常比输入贵 2–3 倍

# 怎么让prompt更好？
## 清晰直接的表达
- 明确的受众语气
- 锁定输出篇幅
- 同时给出要的和不要
- 说明最终用途
- 把复杂任务拆成步骤
## 为AI分配角色
- 专业领域
- 行为方式
- 核心立场
## 用XML标签分离数据与指令
防止恶意prompt
## 控制输出格式
- 方法一：直接描述想要的格式
- 方法二：提供模板让AI填写
- 方法三：预填充
## 让AI逐步思考
- 用标签隔离思考过程
- 直接要求一步步来
- 先列论据再下结论
## 用示例教会AI
- 覆盖典型变体
- 格式完全统一
- 质量高于数量
## 避免AI胡说八道
尤其是医疗法律财务等领域幻觉代价极高
- 明确允许 AI 说我不知道（最简单有效）
- 限制 AI 只使用你提供的信息
- 先找证据，再给结论
- 要求标注置信度
- 降低随机性（API 开发者）
