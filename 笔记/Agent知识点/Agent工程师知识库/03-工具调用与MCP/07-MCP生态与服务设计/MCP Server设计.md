# MCP Server 设计

## 一句话定位

MCP Server 是把外部工具、资源和提示词能力标准化暴露给模型应用的服务层。

## 设计角度

- Tools：可执行动作，如查库、发请求、读文件。
- Resources：可读取资源，如文档、配置、数据库记录。
- Prompts：可复用提示词模板。
- Transport：stdio、HTTP/SSE 等通信方式。
- Auth：访问控制、租户隔离、密钥管理。

## 适合场景

- 希望多个 Agent/客户端复用同一批工具。
- 工具生态增长，需要统一发现和接入规范。
- 企业内部系统很多，需要标准化暴露给 Agent。
