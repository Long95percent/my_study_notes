# python SDK
1️⃣ **本质定义（核心）**  
Python SDK 是对远程服务 **API 的抽象封装层**，把原本基于 HTTP/JSON 的调用，提升为 Python 中的函数/对象调用，本质上是在做 **协议层 → 语言层的映射**，让开发者不用直接处理请求构造、序列化和通信细节。

2️⃣ **技术内核（它真正解决什么问题）**  
一个成熟的 Python SDK 不只是“少写代码”，而是系统性解决 **工程复杂度问题**：统一认证（token/signature）、连接管理（session/keep-alive）、错误模型（异常体系）、重试与限流（retry/backoff）、以及数据结构建模（schema → class），本质是在客户端实现一层 **稳定性与一致性中间层**。

3️⃣ **工程价值（为什么必须用）**  
在实际系统（尤其是 Agent / RAG / 多服务编排）中，SDK充当的是 **调用边界的标准化接口**，把外部不稳定的服务转化为内部可控模块，从而实现解耦、可测试性和可维护性，否则你写的就是一堆分散的 HTTP 调用逻辑，后期几乎不可演进。
