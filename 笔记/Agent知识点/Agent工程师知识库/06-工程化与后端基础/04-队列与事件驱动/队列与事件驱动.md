# 队列与事件驱动

## 一句话定位

队列用于把耗时任务、异步任务和突发流量从主请求链路里拆出去。

## 技术怎么选

| 技术 | 适合场景 | 不适合 |
|---|---|---|
| Kafka | 高吞吐事件流、日志、埋点 | 简单任务队列过重 |
| RabbitMQ | 可靠任务分发、路由灵活 | 极高吞吐日志流 |
| Redis Queue/RQ | 轻量 Python 任务队列 | 复杂可靠性要求 |
| Celery | Python 异步任务、定时任务 | 运维和配置较复杂 |
| SQS/PubSub | 云原生托管队列 | 强本地私有化 |

## Agent 场景

- 文件上传后异步解析和索引。
- 长任务 Agent 后台执行。
- 工具调用日志异步写入分析库。
- 批量评测任务调度。
