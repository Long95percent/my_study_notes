# 结构化输出与 JSON Schema

## 一句话定位

结构化输出是让模型从“会聊天”变成“能被程序稳定消费”的关键。

## 积累角度

- 为什么需要：工具调用、任务计划、分类抽取、评测结果都需要机器可解析。
- 常见格式：JSON、YAML、XML 标签、函数调用参数。
- JSON Schema：字段类型、必填字段、枚举值、数组、嵌套对象。
- 失败处理：解析失败重试、自动修复 JSON、降级为人工审批。
- 安全边界：不能只信模型输出，必须做服务端校验。

## 面试表达

> 我会把模型输出当成“不可信输入”，先用 Schema 约束格式，再在服务端做参数校验和权限校验，最后才进入工具执行层。
