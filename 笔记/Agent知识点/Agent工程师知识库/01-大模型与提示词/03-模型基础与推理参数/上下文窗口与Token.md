# 上下文窗口与 Token

## 一句话定位

上下文窗口决定模型一次推理能看到多少信息，Token 决定输入输出成本、延迟和截断风险。

## 需要积累的角度

- Token 如何计算：中文、英文、代码、JSON 的 Token 密度不同。
- 上下文窗口限制：历史对话、工具结果、RAG 片段都在抢窗口。
- 截断风险：长工具输出、长文档、长计划容易导致关键信息丢失。
- 压缩策略：摘要、滑动窗口、重要性评分、检索式记忆。
- 工程指标：prompt_tokens、completion_tokens、上下文利用率、截断率。
