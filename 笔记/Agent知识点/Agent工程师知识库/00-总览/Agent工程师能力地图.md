# Agent 工程师能力地图

## 1. LLM 基础与 Prompt

- 理解上下文窗口、Token、模型能力边界。
- 能用结构化 Prompt 明确角色、任务、约束、输出格式。
- 能判断什么时候靠 Prompt，什么时候需要工具、记忆或工作流。

## 2. Agent 编排

- 理解 Agent 的感知、规划、执行、观察、反思循环。
- 掌握 ReAct、Plan-and-Execute、状态机、图编排等模式。
- 能用 LangGraph 等框架表达有状态、可恢复、可观测的任务流。

## 3. 工具调用与协议

- 掌握 Tool Calling / Function Calling 的结构化调用机制。
- 理解 ToolRegistry、ToolRuntime、权限白名单、参数校验和错误处理。
- 理解 MCP 如何把外部工具、资源和上下文标准化接入模型应用。

## 4. 记忆系统与 RAG

- 掌握短期记忆、长期记忆、语义记忆、情景记忆的区别。
- 掌握 Chunking、Embedding、向量库、混合检索、重排与边界化召回。
- 能设计记忆写入、更新、遗忘、压缩和反污染机制。

## 5. Multi-Agent 协作

- 理解什么时候需要多 Agent，而不是单 Agent + 工具。
- 掌握角色拆分、任务分解、协作协议、冲突解决和结果汇总。
- 能说明 Supervisor、Swarm、SubAgent 等架构差异。

## 6. 工程化能力

- 能处理并发、异步、超时、重试、幂等、限流、审计和降级。
- 能设计可观测指标：成功率、延迟、成本、工具调用准确率、检索命中率。
- 能把 Agent 系统接入后端服务、网关、SDK 和权限体系。

## 7. 项目表达与面试

- 能讲清项目背景、业务目标、架构设计、核心难点、Trade-off 和结果指标。
- 能把“记忆系统、工具调用、边界化召回、多 Agent”讲成可落地方案。
