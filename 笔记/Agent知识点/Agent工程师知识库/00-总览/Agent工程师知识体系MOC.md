# Agent 工程师知识体系 MOC

> [!tip] 使用方式
> 这页不是让你一次学完，而是告诉你 Agent 工程师应该从哪些角度积累。先看一级能力，再进入具体专题。

## 1. 模型与 Prompt 层

- [[Prompt Engineering]]：让模型理解任务。
- [[上下文窗口与Token]]：决定能塞多少信息、怎么压缩。
- [[结构化输出与JSON Schema]]：让模型输出可解析结果。
- [[Prompt注入与防御]]：防止用户把系统指令绕掉。

## 2. Agent 编排层

- [[LangGraph]]：图式编排、有状态工作流。
- [[ReAct执行模式]]：思考-行动-观察循环。
- [[Plan-and-Execute模式]]：先规划再执行。
- [[Agent状态管理与Checkpoint]]：中断恢复、长任务保存。
- [[Human-in-the-loop审批]]：高风险动作前让人确认。

## 3. 工具与协议层

- [[Tool Calling]]：模型生成工具调用请求。
- [[ToolRegistry]]：工具注册、发现、权限、元数据。
- [[tool_runtime]]：真正执行工具、处理隔离/超时/重试。
- [[mcp协议]]：标准化接入外部工具和上下文。
- [[工具Schema与参数校验]]：防止脏参数和幻觉参数。

## 4. 记忆与 RAG 层

- [[切片Chunking]]：知识库建设入口。
- [[Embedding模型选型]]：语义表示质量。
- [[重排Reranker]]：提升 Top-K 质量。
- [[边界化召回]]：控制召回范围。
- [[存储选型总览]]：数据库/向量库/搜索引擎怎么选。
- [[记忆系统]]、[[如何设计Agent的记忆写入机制]]：长期记忆核心设计。

## 5. 多 Agent 协作层

- [[Multi-Agent，SubAgent等架构]]：架构形态。
- [[Agent协作架构]]：职责拆分与协作。
- [[Agent通信协议]]：消息、上下文、共享状态。
- [[冲突解决与结果汇总]]：多 Agent 输出合并。

## 6. 工程化与生产层

- [[JAVA后端]]：网关、鉴权、限流。
- [[asyncio]]：异步 I/O。
- [[队列与事件驱动]]：Kafka/RabbitMQ/Celery 等。
- [[部署与运行环境]]：Docker/K8s/Serverless。
- [[配置与凭证管理]]：Secret、环境变量、密钥轮换。

## 7. 评测、观测与安全

- [[Agent评测指标体系]]：任务成功率、工具调用准确率、恢复率。
- [[RAG评测指标]]：Recall@K、MRR、nDCG、Faithfulness。
- [[Tracing与可观测性]]：OpenTelemetry、LangSmith、日志链路。
- [[成本延迟与容量规划]]：Token 成本、并发、缓存。
- [[安全治理总览]]：权限、隐私、审计、内容安全。

- [[Agent项目架构模板]]：项目如何讲清楚。
