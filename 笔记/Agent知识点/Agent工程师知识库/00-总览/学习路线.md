# 学习路线

## 第一阶段：能讲清概念

1. Prompt Engineering
2. Tool Calling
3. MCP 协议
4. LangGraph / Agent 编排
5. Chunking / Embedding / Vector DB

## 第二阶段：能做出系统

1. 设计 ToolRegistry 与 ToolRuntime。
2. 设计短期记忆 + 长期记忆 + 检索召回。
3. 设计权限白名单、参数校验、异常恢复。
4. 设计可观测指标与日志链路。
5. 设计一个可演示的端到端 Agent 项目。

## 第三阶段：能应对面试追问

1. 为什么不用纯 Prompt？
2. 为什么需要 LangGraph 这类编排框架？
3. 工具调用如何防止越权和幻觉调用？
4. 长期记忆如何写入、检索、更新和遗忘？
5. RAG 召回不准怎么办？
6. Multi-Agent 什么时候会比单 Agent 更差？
7. 你的项目最大困难是什么，怎么解决？
