# 权限审计与审批

## 一句话定位

权限决定 Agent 能不能做，审计决定出问题后能不能追溯，审批决定高风险动作能不能被人拦住。

## 设计维度

- 主体：用户、Agent、SubAgent、服务账号。
- 客体：工具、数据、文件、数据库、外部 API。
- 动作：读、写、删除、发送、支付、执行代码。
- 条件：时间、租户、环境、风险等级。

## 常见模型

- RBAC：基于角色的权限控制。
- ABAC：基于属性的权限控制。
- Policy Engine：OPA、Casbin。
