# 存储选型总览

## 总体原则

不同存储解决不同问题，不要试图用一个数据库包打天下。

| 类型 | 代表技术 | 适合存什么 | 不适合什么 |
|---|---|---|---|
| 关系型数据库 | PostgreSQL, MySQL | 用户、任务、权限、工具调用记录 | 高维向量海量 ANN |
| 轻量数据库 | SQLite | 本地状态、原型、单机应用 | 高并发写入 |
| 缓存/KV | Redis | 会话状态、短期记忆、锁、限流 | 长期强一致复杂查询 |
| 向量库 | Milvus, Qdrant, Weaviate, Pinecone | Embedding 向量和语义召回 | 复杂事务 |
| 全文检索 | Elasticsearch, OpenSearch | BM25、日志、关键词搜索 | 强事务、低成本小项目 |
| 图数据库 | Neo4j, NebulaGraph | 实体关系、知识图谱、多跳关系 | 简单 CRUD |
| 对象存储 | S3, MinIO, OSS | 原始文件、PDF、图片、离线数据集 | 低延迟小对象频繁更新 |
| 分析数据库 | ClickHouse | 日志、评测结果、埋点分析 | OLTP 事务 |

## Agent 常见组合

- 个人助理原型：SQLite + FAISS + 本地文件。
- 中小型 RAG：PostgreSQL + pgvector + Redis。
- 企业知识库：PostgreSQL + Milvus/Qdrant + Elasticsearch + S3。
- 可观测平台：PostgreSQL + ClickHouse + OpenTelemetry。
