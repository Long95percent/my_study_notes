# 数据摄取与索引管线

## 一句话定位

RAG 的质量很大程度取决于索引前的数据处理，而不是只取决于向量库。

## 典型流程

1. 数据源接入：PDF、网页、Markdown、数据库、聊天记录、工单。
2. 解析清洗：去噪、去重、保留标题层级、抽取元数据。
3. 切片：按语义、标题、长度、重叠窗口切分。
4. Embedding：生成向量。
5. 存储：向量库 + 原文存储 + 元数据索引。
6. 更新：增量索引、删除同步、版本管理。

## 关键元数据

- source_id
- document_id
- chunk_id
- title_path
- created_at / updated_at
- author / tenant_id
- permission_scope
